Project Overview

The Dance Event Application is the most comprehensive and collaborative project I‚Äôve worked on, developed over four months by a six-member team. This project is a web-based dance event management system designed to address inefficiencies in managing dance competitions. The platform not only streamlines participant registration but also enhances competition management for organizers.

With over 14,000 users (parents, teachers, and organizers), this platform provides a unified, automated, and scalable solution for registering and managing dance competitions. From handling eligibility requirements to secure payment processing, this application meets the needs of diverse stakeholders in the dance competition ecosystem.

Confidentiality Notice: Due to a non-disclosure agreement (NDA) signed with our client, we cannot share the source code or sensitive project details. However, we are happy to share an overview of the system and its capabilities.

Key Features

1. User Roles and Permissions

  • Super Organizer:
    • Oversees all competitions, users, and permissions.
    • Manages requests from new organizers.
  • Organizer:
    • Creates and manages competitions, sets eligibility rules (e.g., age, group size).
  • Participant:
    • Registers individuals or groups for competitions.
    • Parents and teachers can manage registrations for their children or students.

2. Participant Registration

  • Allows individual or group-based registrations.
  • Automatically calculates age and group eligibility based on birthdates.

3. Competition Management

  • Organizers can:
    • Create and edit competitions.
    • Set custom rules for age ranges, participant limits, and group sizes.
    • Monitor and analyze registrations and seat availability.

4. Payment Integration

  • Secure multi-competition payment processing using Stripe.
  • Participants can add multiple competitions to a cart and pay in one transaction.

5. Data Analytics

  • Provides organizers with:
    • Insights into registration trends.
    • Data for planning and optimizing future events.

6. Automation and Scalability

  • Automates repetitive tasks like age calculation and eligibility checks.
  • Hosted on AWS to handle high registration volumes during peak times.

Technologies Used

Technology Purpose
React (18.2.0) Frontend development and user interface.
ASP.NET Core (8) Backend API development and business logic.
SQL Server (2022) Database management for structured data.
JWT Secure user authentication and authorization.
AWS Hosting and scalability (API Gateway, Amplify, Lambda).
Stripe Secure payment gateway integration.

System Architecture

The system uses a tiered architecture with the following components:

  1. Frontend (React): A user-friendly interface allowing participants and organizers to manage events and registrations.
  2. Backend (ASP.NET Core): A RESTful API layer for handling business logic, user authentication, and database operations.
  3. Database (SQL Server): Stores user, competition, and registration data.
  4. Authentication (JWT): Ensures secure access to APIs and user-specific data.
  5. Cloud Hosting (AWS): Provides reliability, scalability, and cost-effective infrastructure.

Database Schema

The following Entity-Relationship Diagram (ERD) illustrates the database structure used in the Dance Event Application. It includes relationships between users, participants, groups, competitions, and registrations.

  • Users Table: Stores user information, including roles (Super Organizer, Organizer, Participant).
  • Persons Table: Contains data for individual participants (e.g., children or students).
  • Groups Table: Represents participant groups with calculated age ranges for eligibility.
  • Competitions Table: Holds information about dance competitions, including rules and schedules.
  • PersonCompetition & GroupCompetition Tables: Establish many-to-many relationships between participants/groups and competitions.

Development Journey

1. Planning and Requirement Gathering

  • Collaborated with the client to define functional requirements.
  • Created initial wireframes and system design.

2. Team Collaboration

  • A six-member team with clear role assignments:
    • Frontend: 3 members focused on UI/UX.
    • Backend: 3 members focused on API and database.
  • Weekly meetings to ensure smooth communication and progress tracking.

3. Challenges and Solutions

  • Concurrency Issues: Implemented database-level locks to prevent double bookings.
  • Scalability: Optimized database queries and used AWS services to handle high traffic.
  • Budget Constraints: Managed the project within the $200 AWS credit provided by the school.

4. Deployment

  • Successfully deployed the system on AWS, with a fully functional web interface for organizers and participants.

Planned Enhancements

  • Dance Video Uploads: Allow participants to submit videos for remote judging.
  • Mobile App: Expand accessibility by developing a companion mobile app.
  • Real-Time Notifications: Notify users about registration deadlines and updates.
  • Leaderboards: Add scoring and ranking features for competitions.
